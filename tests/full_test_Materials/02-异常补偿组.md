# [异常补偿组] 事务回滚、状态逆转、最终一致性

## 用例 E2E-EXC-001：入托申请驳回 — 事务回滚后状态不变

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-EXC-001 |
| **前置数据条件** | `{{adminToken}}` 登录；`{{enrollmentId}}` 状态为 PENDING |
| **操作步骤** | **Step 1**: `POST /enrollment/{{enrollmentId}}/review` — 园长审核**驳回**，body: `{"approved":false,"rejectReason":"托位已满"}`<br>**Step 2**: `GET /enrollment/{{enrollmentId}}` — 验证状态 = REJECTED |
| **预期数据库变更** | `Enrollment.status='REJECTED'`，`reject_reason='托位已满'`，`reviewed_by` 非空<br>回滚后：`EnrollmentStatusHistory` 新增 PENDING→REJECTED 记录<br>**反断言**：`Classroom` 的当前使用托位数**不增加** |
| **预期API响应** | Step 1: `{"success":true,"data":{"status":"REJECTED","rejectReason":"托位已满"}}`<br>Step 2: `{"success":true,"data":{"status":"REJECTED"}}` |

## 用例 E2E-EXC-003：暂停入托（ACTIVE→SUSPENDED）→ 考勤自动冻结

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-EXC-003 |
| **前置数据条件** | `{{enrollmentId}}` 状态为 ACTIVE；当日已有 CHECKED_IN 考勤记录；`{{adminToken}}` 登录 |
| **操作步骤** | **Step 1**: `POST /enrollment/{{enrollmentId}}/suspend` — 暂停入托，body: `{"reason":"家长出差，暂停2周"}`<br>**Step 2**: `GET /enrollment/{{enrollmentId}}` — 验证 status = SUSPENDED<br>**Step 3**: `GET /attendance/classroom/{{classroomId}}?date=2026-07-19` — 验证**当日已签到记录不变**<br>**Step 4**: `GET /attendance/classroom/{{classroomId}}?date=2026-07-20` — 验证**未来日期考勤状态 = ABSENT**（冻结）<br>**Step 5**: `GET /attendance/classroom/{{classroomId}}?date=2026-07-21` — 验证**未来日期考勤状态 = ABSENT**<br>**Step 6**: `POST /attendance/check-in` — 尝试签到（SUSPENDED 状态），body: `{"enrollmentId":"{{enrollmentId}}","attendanceDate":"2026-07-20"}` → 应返回失败 |
| **预期数据库变更** | `Enrollment.status='SUSPENDED'`<br>`AttendanceRecord` 中当天之后的无考勤记录日期，自动生成 status='ABSENT' 的记录（懒生成或预生成）<br>**反断言**：SUSPENDED 状态下不可创建新的 CHECKED_IN 考勤 |
| **预期API响应** | Step 1: `{"success":true,"data":{"status":"SUSPENDED"}}`<br>Step 4-5: 该 enrollment 对应考勤记录 status = "ABSENT"<br>Step 6: `{"success":false,"message":"入托状态已暂停，无法签到"}` 或类似 |

| 项目 | 内容 |
|---|---|

| **操作步骤** | **Step 1**: `POST /enrollment/{{enrollmentId}}/reactivate` — 复托，body: `{"reason":"家长已回来"}`<br>**Step 2**: `GET /enrollment/{{enrollmentId}}` — 验证 status = ACTIVE，`updated_at` 更新<br>**Step 3**: 原 SUSPENDED 期间的 ABSENT 记录处理方式验证<br>**Step 4**: `POST /attendance/check-in` — 验证可签到，body: `{"enrollmentId":"{{enrollmentId}}","attendanceDate":"2026-07-22"}`<br>**Step 5**: `GET /attendance/classroom/{{classroomId}}?date=2026-07-22` — 验证 status = CHECKED_IN |

| **预期API响应** | Step 1: `{"success":true,"data":{"status":"ACTIVE"}}`<br>Step 4: `{"success":true,"data":{"status":"CHECKED_IN"}}`<br>Step 5: `{"success":true,"data":[{"status":"CHECKED_IN"}]}` |
