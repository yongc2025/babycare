# [权限穿透组] 跨角色 RBAC 穿透校验与数据隔离

## 用例 E2E-RBAC-001：删除入托档案 — 跨角色权限穿透（核心 RBAC 用例）

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-RBAC-001 |
| **前置数据条件** | `{{enrollmentId}}` 状态为 PENDING（刚创建，可删除）；提前获取 7 种角色的 Token：`{{parentToken}}`, `{{teacherToken}}`, `{{caregiverToken}}`, `{{healthToken}}`, `{{financeToken}}`, `{{safetyToken}}`, `{{opsToken}}`, `{{admToken}}`, `{{adminToken}}`、`{{directorToken}}` |
| **操作步骤** | **Step 1**: 用 `{{parentToken}}` 调用 `DELETE /enrollment/{{enrollmentId}}`（或等效删除接口）→ 预期 **403**<br>**Step 2**: 用 `{{teacherToken}}` 调用 → 预期 **403**<br>**Step 3**: 用 `{{caregiverToken}}` 调用 → 预期 **403**<br>**Step 4**: 用 `{{healthToken}}` 调用 → 预期 **403**<br>**Step 5**: 用 `{{financeToken}}` 调用 → 预期 **403**<br>**Step 6**: 用 `{{safetyToken}}` 调用 → 预期 **403**<br>**Step 7**: 用 `{{opsToken}}` 调用 → 预期 **403**<br>**Step 8**: 用 `{{admToken}}` 调用 → 预期 **403**<br>**Step 9**: 用 `{{directorToken}}`（园长）调用 → 预期 **200/成功**<br>**Step 10**: 用 `{{adminToken}}`（系统管理员）调用 → 预期 **200/成功** |
| **预期数据库变更** | Step 1-8 后：**无变更**，`Enrollment` 表记录完全不受影响<br>Step 9 或 Step 10 后：`Enrollment` 记录被删除（或标记 deleted） |
| **预期API响应** | Step 1-8: `{"success":false,"status":403,"message":"权限不足"}`<br>Step 9-10: `{"success":true,"message":"删除成功"}` |

## 用例 E2E-RBAC-002：查看机构数据 — 园长只看本园，老板看多园，教师只看本班

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-RBAC-002 |
| **前置数据条件** | 已创建 2 个机构：`{{orgId}}`（机构A）和 `{{orgIdB}}`（机构B）；机构A有班级 `{{classroomId}}`，机构B有班级 `{{classroomIdB}}`；教师 `{{teacherToken}}` 只分配到机构A的班级 |
| **操作步骤** | **Step 1**: 园长 `{{directorToken}}` 调用 `GET /organization/{{orgId}}` → 成功查看本园详情<br>**Step 2**: 园长 `{{directorToken}}` 调用 `GET /organization/{{orgIdB}}` → 预期 **403**（不可越园）<br>**Step 3**: 教师 `{{teacherToken}}` 调用 `GET /organization/{{orgId}}` → 预期 **403**（教师不可查看机构管理）<br>**Step 4**: 教师 `{{teacherToken}}` 调用 `GET /attendance/classroom/{{classroomId}}?date=2026-07-19` → 成功（本班数据）<br>**Step 5**: 教师 `{{teacherToken}}` 调用 `GET /attendance/classroom/{{classroomIdB}}?date=2026-07-19` → 预期 **403**（不可看非本班）<br>**Step 6**: 家长 `{{parentToken}}` 调用 `GET /enrollment/my-enrollments` → 只返回自己宝宝数据<br>**Step 7**: 家长 `{{parentToken}}` 调用 `GET /organization/{{orgId}}` → 预期 **403** |
| **预期数据库变更** | 无变更（只读查询） |
| **预期API响应** | Step 2,3,5,7: `{"success":false,"status":403}`<br>Step 1,4,6: `{"success":true,"data":{...}}` |

## 用例 E2E-RBAC-003：日报操作权限隔离 — 教师编辑/园长审核/家长只读

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-RBAC-003 |
| **前置数据条件** | `{{dailyReportId}}` 状态为 DRAFT；`{{parentToken}}`、`{{teacherToken}}`、`{{adminToken}}`（园长）均已登录 |
| **操作步骤** | **Step 1**: 家长 `{{parentToken}}` 调用 `PUT /daily-report/{{dailyReportId}}` → 预期 **403**<br>**Step 2**: 家长 `{{parentToken}}` 调用 `POST /daily-report/{{dailyReportId}}/publish` → 预期 **403**<br>**Step 3**: 教师 `{{teacherToken}}` 调用 `PUT /daily-report/{{dailyReportId}}` → 成功（可编辑）<br>**Step 4**: 教师 `{{teacherToken}}` 调用 `POST /daily-report/{{dailyReportId}}/publish` → 成功（可发布）<br>**Step 5**: 园长 `{{adminToken}}` 调用 `POST /daily-report/{{dailyReportId}}/approve` → 成功（可审核）<br>**Step 6**: 家长 `{{parentToken}}` 调用 `GET /daily-report/baby/{{babyId}}?date=2026-07-19` → 成功（可查看） |
| **预期数据库变更** | 无变更（仅权限校验） |
| **预期API响应** | Step 1-2: `{"success":false,"status":403}`<br>Step 3-5: `{"success":true}`<br>Step 6: `{"success":true,"data":{...}}` |

|---|---|

## 用例 E2E-RBAC-007：无 Token / 错误 Token / 过期 Token 访问受保护接口

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-RBAC-007 |
| **前置数据条件** | `{{enrollmentId}}` ACTIVE |
| **操作步骤** | **Step 1**: 不传 Authorization 头调用 `GET /enrollment/{{enrollmentId}}` → 预期 **401/403**<br>**Step 2**: 传入无效 Token `"Bearer invalid_token_xxx"` → 预期 **401/403**<br>**Step 3**: 传入格式错误 Token `"NotABearer token"` → 预期 **401/403** |

|---|---|

| **操作步骤** | **Step 1**: 家长A `{{parentToken}}` 调用 `GET /enrollment/my-enrollments` → 只返回自己的入托档案<br>**Step 2**: 家长A `{{parentToken}}` 调用 `GET /enrollment/{{enrollmentIdB}}`（家长B的入托）→ 预期 **403**<br>**Step 3**: 家长A `{{parentToken}}` 调用 `GET /daily-report/baby/{{babyIdB}}`（家长B的宝宝日报）→ 预期 **403** |

## 用例 E2E-RBAC-009：邀请码绑定 — 未授权家长不可通过搜索绑定宝宝

| 项目 | 内容 |
|---|---|
| **用例ID** | E2E-RBAC-009 |
| **前置数据条件** | 园长已生成邀请码 `{{inviteCode}}` 绑定 `{{enrollmentId}}`；第三方家长 `{{unboundParentToken}}` 登录 |
| **操作步骤** | **Step 1**: 第三方家长用错误邀请码 `POST /enrollment/bind-by-code`，body: `{"code":"WRONG_CODE"}` → 失败<br>**Step 2**: 第三方家长用正确邀请码 `POST /enrollment/bind-by-code`，body: `{"code":"{{inviteCode}}"}` → 成功<br>**Step 3**: 第三方家长再次调用 `GET /enrollment/my-enrollments` → 现在能看到了 |
| **预期数据库变更** | Step 2 后：`EnrollmentGuardian` 新增 1 条记录，`user_id` 为第三方家长 |
| **预期API响应** | Step 1: `{"success":false,"message":"无效的邀请码"}`<br>Step 2: `{"success":true,"data":{"enrollmentId":"{{enrollmentId}}"}}` |
