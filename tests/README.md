# BabyCare/HuiGrowth 测试套件

本目录包含四套完整的测试案例，用于验证系统的功能正确性、性能、安全防护和用户业务流程。

## 📋 测试结构

```
tests/
├── README.md                          # 本文件
├── requirements.txt                   # Python 依赖
├── conftest.py                        # 共享基础设施（API 客户端、测试账号、结果收集）
├── run_all_tests.py                   # 全量测试运行器
├── test_functional_integration.py     # 🔗 功能联调测试
├── test_performance.py                # ⚡ 性能测试
├── test_security.py                   # 🛡️ 安全测试
└── test_acceptance.py                 # 🧪 验收测试
```

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r tests/requirements.txt
```

依赖：仅需要 `requests` 库。

### 2. 确保服务已启动

```bash
# 方式一：完整启动
start-dev.bat

# 方式二：分步启动
# 终端1 - 后端
cd backend
call mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 终端2 - 前端（部分测试需要前端配合）
cd frontend
npm run dev
```

### 3. 运行测试

```bash
# 运行全部测试
python tests/run_all_tests.py

# 运行单个模块
python tests/test_functional_integration.py
python tests/test_performance.py
python tests/test_security.py
python tests/test_acceptance.py
```

## 📊 测试模块说明

| 4. 考勤模块 | 签到、签退、请假申请/审批 | ~4 |
| 5. 照护记录 | 喂养/睡眠记录、查询 | ~4 |
| 6. 健康观察 | 晨检创建、班级查询 | ~2 |
| 7. 日报模块 | 生成、更新、发布、家长查看 | ~4 |
| 8. 食谱 | 创建、发布、机构查询 | ~3 |
| 9. 安全台账 | 创建、处理、关闭、列表 | ~4 |
| 10. 通知公告 | 创建、发布、列表 | ~3 |
| 11. 招生线索 | 创建、列表 | ~2 |
| 12. 园长驾驶舱 | 运营概览、工作台 | ~2 |

| 14. 入托状态变更 | 暂停、复托、历史 | ~3 |

### ⚡ 性能测试 (`test_performance.py`)

测量关键 API 端点的响应时间和并发能力。

| 章节 | 测试内容 | 阈值 |
|------|----------|------|
| 1. 单请求延迟 | 健康检查、登录、机构、驾驶舱 | 快速<0.3s 中等<1s 复杂<3s |
| 2. 并发请求 | 健康检查10并发、机构查询10并发 | 最大<5s |
| 3. 分页性能 | 员工、通知、台账、线索列表 | 快速<0.3s |
| 4. 写入吞吐量 | 连续创建通知(5次)、连续读取(5次) | 平均<1s |
| 5. 长耗时接口 | 财务工作台、营养分析、日报生成 | 复杂<3s |

### 🛡️ 安全测试 (`test_security.py`)

验证身份认证、权限控制和输入安全防护。

| 章节 | 测试内容 | 说明 |
|------|----------|------|
| 1. 认证绕过 | 6个受保护端点无 token 访问 | 应返回 401/403 |
| 2. Token 安全 | 空 token、伪造 JWT、特殊字符、超长 token | 应全部拒绝 |
| 3. 权限隔离 | 教师越权管理操作、家长越权查入托 | 应拒绝 |
| 4. SQL 注入 | 路径参数、body、查询参数注入 | 应正确过滤 |
| 5. XSS | 多种 XSS payload 存储测试 | 应转义/拒绝 |
| 6. 参数篡改 | 超长名称、负数 ID、无效 JSON、错误方法 | 应校验 |
| 7. 信息泄露 | 错误响应含堆栈、登录差异提示 | 不应泄露 |
| 8. 暴力破解 | 连续失败登录频率限制检查 | 应有限流 |

### 🧪 验收测试 (`test_acceptance.py`)

从用户视角验证核心业务流程。

| 场景 | 测试内容 | 角色 |
|------|----------|------|
| 1. 新宝宝入托全流程 | 招生→报名审核→分班→入托档案 | ADMIN |
| 2. 一日照护流程 | 考勤→照护(喂养+睡眠)→签退→日报 | TEACHER |
| 3. 家长端体验 | 个人信息→家庭→宝宝→成长记录→日报→申请 | PARENT |
| 4. 园长驾驶舱 | 运营概览→工作台→安全统计→招生漏斗→员工 | ADMIN |
| 5. 机构运营流程 | 通知→食谱→安全台账→监管报表 | ADMIN |

## 🔧 配置说明

所有配置在 `conftest.py` 中：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `BASE_URL` | `http://localhost:8080/api` | 后端 API 地址 |
| `REQUEST_TIMEOUT` | 15s | 请求超时时间 |
| `TEST_ACCOUNTS` | admin/teacher/parent | 测试账号 |

## 📌 注意事项

1. **服务必须已启动**：测试需要后端运行在 `localhost:8080`。
2. **测试账号需存在**：默认使用 T040 创建的测试账号。
3. **测试有副作用**：会创建真实的测试数据（通知、照护记录等），建议在开发/测试环境运行。
4. **性能测试受环境影响**：本地开发环境的结果仅供参考，不应作为正式性能基准。
5. **Windows 兼容**：所有脚本在 PowerShell 和 CMD 下均可运行。
