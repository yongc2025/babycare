# BabyCare/HuiGrowth 项目宪法

本文件是 AI 编码助手进入项目时的根入口。它定义最高优先级原则和文档路由；具体细则以 `docs/` 下的专项文档为准。

如果本文与专项文档重复，以专项文档为主；如果文档与代码事实冲突，以代码事实为准，并在本轮任务中同步更新文档。

## 1. 项目定位

BabyCare/HuiGrowth 是托育 + 保育机构运营平台。

当前阶段：

- 后端：Spring Boot 模块化单体。
- 前端：单 Web 多角色。
- 目标：优先形成可验证的托育保育业务闭环，不提前陷入微服务、多端、硬件和监管外部对接的完整复杂度。

## 4. 核心工程规则

> 以下规则以 `docs/01-开发规范/编码规范.md` 为唯一真源。本文仅列出摘要。

- **复用/扩展/新建顺序**：新增能力前按「复用已有 → 扩展现有 → 新建」判断（编码规范.md §1.1）。
- **真实数据流**：新功能必须有真实数据流，不得新增 mock fallback（编码规范.md §6）。
- **抽象原则**：抽象必须服务真实业务边界、稳定复用点或明确复杂度（编码规范.md §1.1）。
- **任务聚焦**：每任务只完成一个明确业务闭环，避免大面积无关 diff。
- **Windows 兼容**：所有命令必须可在本仓库 Windows 开发环境中执行。

## 5. 编码

- 所有源代码文件和注释必须使用 UTF-8 编码。
- 如果中文内容疑似乱码，必须用 `node` 或 `rg` 验证真实 UTF-8 内容，不只相信 PowerShell 的显示结果。
- 不得在乱码文件上继续叠加功能。
