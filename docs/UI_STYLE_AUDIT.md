# UI 样式审计与改造方向

最近更新：2026-07-15

## 1. 审计结论

当前前端已从早期 demo/template 风格开始向托育机构工作台风格迁移，但仍处于第一轮视觉骨架阶段。

已完成：

- 全局 CSS token 和 Ant Design theme 初步统一。
- 登录页重构为托育机构系统入口。
- 主布局重构为后台工作台结构。
- Dashboard 改为“今日托育工作台”。
- 核心业务页先恢复为清晰中文骨架。

仍需继续：

- 截图级视觉验收。
- 注册页与登录页统一。
- 核心业务页从骨架升级为真实交互。
- 抽取通用页面组件，减少内联样式。
- 将导航从家庭育儿产品进一步改为机构托育主链。

## 2. 目标风格

托育 + 保育系统应体现：

- 安全可信
- 专业有序
- 温和亲切
- 适合高频操作
- 信息密度适中
- 机构端清晰高效，家长端温暖可信

应避免：

- 过度儿童化
- 大面积紫色/橙色渐变
- 营销落地页风格
- 过多装饰图形
- 过重阴影
- 卡片套卡片

## 3. 推荐视觉方向

建议采用“现代托育机构运营系统”风格：

| 用途 | 建议色值 | 说明 |
|---|---|---|
| Primary | `#167D7F` | 稳定、健康、机构感 |
| Primary Hover | `#0F6B6D` | 主按钮 hover |
| Accent | `#F5A524` | 温暖提醒，不做大面积主色 |
| Background | `#F5F7FA` | 应用背景 |
| Surface | `#FFFFFF` | 内容容器 |
| Border | `#E5EAF0` | 分割线 |
| Text Primary | `#1F2933` | 主文本 |
| Text Secondary | `#64748B` | 次级文本 |
| Success | `#2F9E44` | 正常/完成 |
| Warning | `#F59F00` | 待处理/提醒 |
| Error | `#E03131` | 异常/错误 |
| Info | `#2563EB` | 信息状态 |

## 4. 页面级改造方向

### 登录/注册

当前登录页已完成第一轮重构。后续需要：

- 注册页与登录页保持同一视觉语言。
- 未实现的第三方登录弱化或标记为规划中。
- 补齐忘记密码路由或隐藏入口。
- 做移动端截图验收。

### 主布局

当前主布局已从大白卡内容容器改为工作台布局。后续需要：

- 导航逐步替换为机构、班级、考勤、保育记录、日报、家园沟通。
- Header 接入真实机构/班级上下文。
- 通知数量改为真实接口或隐藏。

### Dashboard

当前 Dashboard 已改为“今日托育工作台”。后续需要：

- 指标来自真实接口。
- 展示今日应到、已到、请假、异常、待发布日报。
- 班级概览和待办事项接入真实数据。
- 无数据时展示明确空状态，不展示假数据。

### 业务页面

AI 育儿、保育记录、教育规划、家园协作、家庭管理目前是干净中文骨架。后续需要：

- 抽取 PageHeader、MetricCard、StatusTag、EmptyState 等通用组件。
- 减少内联样式。
- 按真实托育主链逐步接 API。
- 未完成能力显示“待接入”，不得假装可用。

## 6. 验收标准

视觉改造任务完成时必须满足：

- 页面文案无乱码。
- 前端 `npm run build` 通过。
- 登录页首屏符合托育 + 保育系统定位。
- 主色、背景、按钮、卡片、导航风格统一。
- 无大面积紫色/橙色渐变。
- 卡片圆角以 6px-8px 为主。
- 不出现整体卡片套卡片布局。
- 移动端登录页不溢出。
- 关键页面完成截图级验收。
