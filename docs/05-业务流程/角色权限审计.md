# 角色工作内容与权限边界定义

最近更新：2026-07-19

## 1. 角色体系概览

系统采用三层角色体系，每层独立但协同工作：

| 层级 | 角色/岗位 | 说明 |
|---|---|---|
| 登录用户角色 | `ADMIN`, `PARENT`, `ELDER` | 决定系统身份和登录入口，`User.UserRole` 枚举 |
| 机构员工岗位 | `DIRECTOR`, `TEACHER`, `CAREGIVER`, `HEALTH_WORKER`, `HEALTH_DOCTOR`, `FINANCE`, `SAFETY_OFFICER`, `LOGISTICS_STAFF`, `OPERATIONS_STAFF`, `ADMISSIONS_OFFICER` | 决定机构内职责，`Staff.StaffRole` 枚举 |
| 家庭成员角色 | `CREATOR`, `PARENT`, `GRANDPARENT`, `RELATIVE` | 用于家庭内部协作，不等同于系统登录权限 |

设计原则：

```text
登录角色决定系统身份
员工岗位决定机构内职责
组织关系决定数据范围
页面按钮决定操作边界
审计日志记录关键责任
```

## 2. 建议角色模型

### 2.1 平台/系统管理员

定位：维护系统运行、账号、权限和基础配置，不参与具体园区日常业务。

工作内容：

- 初始化系统配置、品牌信息、数据字典。
- 创建平台管理员或重置关键账号。
- 管理全局角色、菜单、按钮权限。
- 查看所有机构/园区基础状态。
- 配置短信、文件上传、通知、登录策略。
- 查看审计日志、异常日志和数据健康状态。
- 处理数据修复、字符集、脏数据、账号冻结等系统问题。

权限边界：

- 可查看全平台配置和机构清单。
- 可创建/禁用平台账号。
- 原则上不直接修改宝宝日常照护记录、家长日报、收费明细，除非进入数据维护流程。

责任边界：

- 对系统可用性、权限配置、数据安全负责。
- 不对单个园区运营结果、宝宝照护过程负责。



### 2.2 老板/机构管理员

定位：机构主体或集团负责人，关注经营结果、风险、资源和多园区统筹。

工作内容：

- 创建和维护机构主体信息。
- 创建多个园区。
- 给每个园区分配园长。
- 查看多园区经营数据。
- 对比园区托位、出勤、招生、收费、健康、安全、师生比。
- 查看异常预警和监管材料完整度。

需要看的内容：

- 园区数、班级数、总托位、已用托位、空余托位。
- 今日出勤率、请假率、异常健康记录。
- 本月新增入托、本月退托、招生线索转化。
- 应收、已收、欠费、退费。
- 安全事故、未关闭台账、监管缺失字段。
- 各园区排名和趋势对比。

权限边界：

- 可看自己机构主体下所有园区。
- 可创建园区、任命园长、查看多园区汇总。
- 不应直接替教师修改每日照护过程，除非具备审计原因。

责任边界：

- 对经营、合规、资源配置、园区负责人任命负责。
- 不对班级每日具体记录的真实性承担第一责任。



### 2.3 园长

定位：单个园区实际负责人，对本园区运营、人员、班级、入托、风险和监管负责。

工作内容：

- 维护本园区资料和监管备案字段。
- 创建班级和托位容量。
- 创建/邀请员工。
- 分配员工岗位和班级。
- 审核宝宝入托、退托、转班。
- 查看本园区出勤、日报、健康、安全、收费、招生、监管。
- 处理未完成日报、异常健康、安全事故、欠费、满班等风险。

权限边界：

- 可管理本园区机构资料、班级、员工、入托档案。
- 可查看本园区全部宝宝和业务记录。
- 可审核关键流程，但不应替老师批量伪造日常照护记录。
- 不可查看其他园区数据，除非同时拥有老板/机构管理员身份。

责任边界：

- 对本园区运营结果、人员配置、班级容量、入托审核、异常闭环负责。
- 对教师和保育员的每日记录进行监督，不是每条记录的第一记录人。



### 2.4 教师

定位：班级教育与家园沟通负责人，负责班级日常观察、教学发展、日报内容。

工作内容：

- 查看自己负责班级和宝宝。
- 记录到园、离园、请假辅助处理。
- 记录活动、情绪、发展观察。
- 编辑并发布家长日报。
- 查看家长反馈、通知回执。
- 参与成长评估、教育计划。

权限边界：

- 只能看自己负责班级或被授权宝宝。
- 可创建本班考勤、照护、健康观察、日报、发展评估。
- 不应创建园区、修改财务、修改监管备案、任命员工。

责任边界：

- 对班级记录、日报内容、教育发展观察负责。
- 对食品安全、财务收款、园区合规不承担主责。



### 2.5 保育员

定位：宝宝一日生活照护第一记录人。

工作内容：

- 记录喂养、饮水、睡眠、如厕、体温、情绪、活动。
- 配合晨午检和全日观察。
- 发现异常时上报教师/保健员。
- 查看本班今日任务和未完成照护记录。

权限边界：

- 只能操作本班照护记录。
- 可新增和补充照护记录。
- 不应发布日报、修改机构资料、处理财务、关闭安全事故。

责任边界：

- 对照护记录的及时性和真实性负责。
- 对健康诊断、监管报表、收费不承担主责。



### 2.6 保健员/保健医

定位：园区卫生保健、健康异常、用药、传染病防控和营养膳食负责人。

工作内容：

- 查看晨午检、全日观察和异常记录。
- 处理健康异常和跟进记录。
- 管理过敏、用药委托、用药执行。
- 管理传染病上报、隔离、复园。
- 管理食谱、营养分析、食品留样。
- 输出卫生保健台账和监管材料。

权限边界：

- 可查看本园区健康、用药、过敏、食谱和相关台账。
- 可处理健康异常和用药审核。
- 不应修改财务、招生、机构经营数据。

责任边界：

- 对健康安全专业判断、用药执行、食品卫生、传染病防控负责。
- 不对招生转化、收费、日常教学评价负责。



### 2.7 安全/后勤负责人

定位：设施安全、消防、消毒、食品留样、环境卫生、隐患闭环负责人。

工作内容：

- 新增和维护安全卫生台账。
- 跟进隐患、事故、设施巡检。
- 维护消毒、食品留样、消防检查记录。
- 关闭处理完成的台账。

权限边界：

- 可操作安全卫生台账和事故跟进。
- 可查看关联事故与园区安全统计。
- 不应操作宝宝日报、财务账单、招生线索。

责任边界：

- 对安全卫生记录和隐患闭环负责。
- 对医疗专业判断、教学记录不承担主责。



### 2.8 财务/运营

定位：收费、账单、招生线索、试托转化、经营效率负责人。

工作内容：

- 配置收费项目。
- 创建账单、标记支付、取消账单。
- 管理招生线索、报名审核、试托、转入托。
- 查看欠费、收款、招生转化、托位使用率。

权限边界：

- 可操作收费、账单、招生线索。
- 可查看必要的宝宝和入托信息。
- 不应修改健康诊断、日常照护原始记录、机构系统配置。

责任边界：

- 对收费准确性、招生推进、经营数据负责。
- 对宝宝日常照护真实性不承担主责。



### 2.9 家长

定位：宝宝监护人，查看宝宝在园情况并与园区协同。

注册与激活策略：

- 家长可以通过公开注册入口创建账号，但公开注册只能创建 `PARENT`。
- 家长注册建议以手机号为主，用户名为兼容字段，后续逐步弱化用户名。
- 正式环境应启用短信验证码；当前开发阶段可以先保留密码注册。
- 微信登录/微信手机号授权适合作为后续增强，不作为第一阶段唯一入口。
- 园长或招生/运营可以在后台生成邀请链接/邀请码，用于把家长绑定到指定宝宝或入托档案。
- 没有园区邀请时，家长只能创建家庭空间和宝宝基础资料，不能看到园区、班级和入托数据。

工作内容：

- 查看宝宝日报、考勤、照护、健康观察。
- 查看通知公告并确认。
- 管理接送人和委托接送。
- 提交请假、用药委托。
- 查看账单和缴费状态。
- 与园区进行家园沟通。

权限边界：

- 只能查看自己家庭/宝宝相关数据。
- 可提交家长侧申请和确认。
- 不可查看其他宝宝、其他班级、园区经营数据。

责任边界：

- 对家庭资料、接送授权、请假、用药委托真实性负责。
- 不对园区照护记录的创建负责。



### 2.10 长辈/其他监护人

定位：低操作门槛的信息查看者。

工作内容：

- 大字版查看日报摘要、到离园、吃睡健康状态。
- 查看接送提醒和重要通知。
- 可根据授权确认部分通知。

权限边界：

- 默认只读。
- 是否允许接送确认、请假、用药，应由家长或园区授权。
- 不应操作园区管理、财务、日报发布。

责任边界：

- 对本人确认的接送/通知行为负责。
- 不对宝宝资料维护和园区记录负责。



## 3. 入托责任边界

宝宝入托不是单一动作，建议拆成流程：

```text
招生/运营录入线索
-> 家长补充宝宝和监护人资料
-> 园长审核托位、班级和入园日期
-> 保健员补充健康、过敏、特殊照护信息
-> 园长确认入托生效
-> 老师/保育员开始日常记录
```

### 3.1 家长注册与宝宝关联决策

家长账号、宝宝资料和入托档案应分开设计：

```text
User 家长账号
-> Family 家庭
-> Baby 宝宝基础资料
-> Enrollment 入托档案
-> Organization/Classroom 园区和班级
```

推荐注册方式：

| 场景 | 注册/激活方式 | 说明 |
|---|---|---|
| 家长主动了解系统 | 手机号 + 短信验证码注册 | 只创建家长账号和家庭空间，不自动绑定园区 |
| 园区已录入宝宝 | 园长/招生生成邀请链接或邀请码 | 家长注册后通过邀请码绑定宝宝和入托档案 |
| 家长先提交报名 | 手机号注册后填写报名/宝宝资料 | 进入招生线索，由园区审核后转入托 |
| 微信生态使用 | 微信登录/手机号授权 | 后续增强，适合家长端和长辈端 |
| 员工账号 | 后台创建或邀请激活 | 不允许公开注册成员工 |

第一阶段建议：

```text
家长公开注册：手机号/用户名 + 密码，后续补短信验证码
园区绑定：园长/招生在后台生成邀请码
宝宝信息：可以由园长建档，也可以由家长补充，最终由园长确认入托
```

正式产品建议：

```text
手机号作为主账号
短信验证码用于注册、找回密码和关键绑定
微信登录作为便捷登录
邀请码用于园区、宝宝、入托档案的安全绑定
```

宝宝信息创建责任：

| 信息类型 | 推荐创建/维护角色 | 说明 |
|---|---|---|
| 宝宝姓名、性别、生日 | 家长或园长/招生 | 家长填写更准确，园区可代录 |
| 监护人、联系方式 | 家长 | 园区可审核 |
| 紧急联系人、接送人 | 家长 | 园长/老师只审核和查看 |
| 过敏、病史、特殊照护 | 家长填写，保健员审核 | 涉及健康风险，需专业角色确认 |
| 入园日期、班级、状态 | 园长 | 属于园区管理责任 |
| 招生线索、试托记录 | 招生/运营 | 园长最终确认入托 |

绑定关系建议：

```text
园长/招生创建或确认 Enrollment
-> 系统生成 inviteCode / invitationLink
-> 家长用手机号注册或登录
-> 输入邀请码
-> 系统创建 FamilyMember 和 EnrollmentGuardian 绑定
-> 家长获得该宝宝的日报、考勤、照护、请假、接送、用药等权限
```

安全边界：

- 家长不能通过搜索宝宝姓名直接绑定宝宝。
- 邀请码应有有效期、使用次数和绑定范围。
- 一个宝宝可绑定多个监护人，但至少一个主监护人。
- 老师只能查看本班宝宝和已授权家长联系方式。
- 长辈默认由家长邀请，权限低于主监护人。

简化版本：

```text
园长创建入托档案
-> 分配班级
-> 绑定家长
-> 录入紧急联系人和健康备注
-> 生效
```

系统实现：

- `Enrollment` 全流程：PENDING [待园长审核] → HEALTH_CHECK [园长通过，待保健审核] → ACTIVE [保健通过，正式入托]；可驳回、暂停、复托、退托。
- `EnrollmentGuardian` 实现家长与入托档案的正式绑定（支持机构直接绑定、邀请码绑定、自助注册绑定）。
- 邀请码机制：园长生成邀请码 → 家长输入加入。
- 手机号+验证码注册，微信登录占位预留。
- 招生线索（`AdmissionLead`）可选择转入托。

## 4. 菜单与权限现状

当前前端菜单已由 `User.role` 简单区分升级为 RBAC 动态角色过滤：

- `MainLayout` 根据 `userInfo.role`（User.UserRole 枚举）和 `staffInfo.staffRole`（Staff.StaffRole 枚举）动态过滤菜单。
- 系统管理后台已支持角色管理、权限管理、菜单管理、用户管理（RBAC 体系）。
- 前端菜单定义含 `roles?: UserRole[]` 字段，支持按角色过滤。
- 员工菜单通过 StaffRole 区分（园长、教师、保育员、保健员、安全/后勤、财务/运营等）。

| 菜单 | 可见角色 | 说明 |
|---|---|---|
| 今日工作台/Dashboard | ADMIN, PARENT, DIRECTOR, TEACHER, CAREGIVER | 内容按角色变化：园长看机构概览，教师/保育员看班级照护，家长看宝宝动态 |
| 园所运营/Organization | ADMIN, DIRECTOR | 机构/班级/员工/入托管理 |
| 班级照护/TeacherWorkbench | TEACHER, CAREGIVER | 一日照护记录、照护任务、漏记提醒 |
| 日报管理/DailyReport | DIRECTOR, TEACHER | 日报发布审核（草稿→待审→已发布） |
| 健康安全/HealthSafety | DIRECTOR, HEALTH_WORKER, HEALTH_DOCTOR, SAFETY_OFFICER, LOGISTICS_STAFF | 保健员/安全后勤工作台 |
| 运营监管/OperationsRegulatory | DIRECTOR, FINANCE, OPERATIONS_STAFF, ADMISSIONS_OFFICER | 招生、收费、食谱、监管导出 |
| 园长工作台/PrincipalWorkbench | DIRECTOR | 待办审批、风险预警、机构概况 |
| 成长记录/GrowthRecord | PARENT | 家长看自己宝宝的成长记录 |
| 家长日报/ParentReports | PARENT, DIRECTOR, TEACHER | 家长看日报，教师/园长可管理 |
| 家庭管理/FamilyManagement | PARENT | 管理家庭成员、长辈权限 |
| 长辈模式/ElderMode | ELDER, PARENT | 大字高对比模式，老年人看宝宝 |
| AI 育儿/AIParenting | PARENT | AI 育儿建议 |
| 教育规划/EducationPlanning | PARENT, DIRECTOR, TEACHER | 教育计划和评估 |
| 家园协作/FamilyCollaboration | PARENT, TEACHER | 家园沟通 |
| 系统管理/SystemManagement | ADMIN | 角色/权限/菜单/用户/审计日志/数据字典/系统配置 |
| 老板驾驶舱/BossDashboard | ADMIN（老板/机构管理员） | 多园区聚合指标对比 |
| 家长申请/ParentApplications | PARENT | 请假/用药/接送申请 |
| 财务/Billing | FINANCE, ADMIN | 收费统计、账单管理 |
| 个人中心/Profile | ALL | 个人资料和设置 |
