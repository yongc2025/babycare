# 系统运转工作流程

最近更新：2026-07-19

## 1. 总体定位

本系统不是单一“家长记录工具”，而是托育保育机构从开通、建园、招生、入托、照护、家园共育、健康安全、收费、监管到经营复盘的协同系统。

完整运转链路：

```text
平台开通
-> 机构/集团入驻
-> 多园区搭建
-> 园区初始化
-> 招生报名
-> 宝宝入托
-> 日常照护和教学
-> 家长同步和协作
-> 健康安全和保健
-> 财务收费
-> 监管和经营分析
-> 持续优化
```

## 2. 组织层级

推荐组织模型：

```text
平台 Platform
└── 机构主体 / 集团 Tenant
    ├── 园区 Campus / Organization A
    │   ├── 班级 Classroom
    │   ├── 员工 Staff
    │   ├── 入托档案 Enrollment
    │   └── 日常业务记录
    └── 园区 Campus / Organization B
        ├── 班级 Classroom
        ├── 员工 Staff
        ├── 入托档案 Enrollment
        └── 日常业务记录
```


## 3. 新上线完整流程

### 3.1 平台初始化

参与角色：平台/系统管理员。

工作流程：

```text
配置系统基础参数
-> 创建平台管理员
-> 配置角色权限
-> 配置数据字典
-> 配置短信/文件/通知/登录策略
-> 开启机构入驻
```

需要完成的功能：

- 系统设置。
- 平台账号管理。
- RBAC 权限配置。
- 数据字典。
- 审计日志。
- 系统健康检查。


### 3.2 机构/集团入驻

参与角色：平台管理员、老板/机构管理员。

工作流程：

```text
平台管理员创建机构主体
-> 创建老板/机构管理员账号
-> 老板登录
-> 补充机构主体资料
-> 创建一个或多个园区
```

老板需要管理：

- 机构主体名称、营业资质、联系人。
- 多个园区。
- 每个园区负责人。
- 多园区经营数据。


### 3.3 多园区搭建

参与角色：老板/机构管理员、园长。

工作流程：

```text
老板创建园区
-> 分配园长
-> 园长接管园区
-> 园长补充园区资料
-> 创建班级
-> 配置托位和月龄范围
-> 创建/邀请员工
-> 分配员工岗位和班级
```

老板关注：

- 哪些园区已开通。
- 哪个园区谁负责。
- 各园区托位、员工、招生和风险情况。

园长关注：

- 本园区资料是否完整。
- 班级是否创建。
- 员工是否到位。
- 班级容量和师生比是否合理。


### 4.1 招生报名

参与角色：招生/运营、家长、园长。

工作流程：

```text
家长咨询或扫码报名
-> 招生/运营创建线索
-> 跟进沟通
-> 安排参观/试托
-> 园长审核托位和班级
-> 确认是否转入托
```


### 4.2 宝宝资料创建

参与角色：家长、招生/运营、园长、保健员。

工作流程：

```text
家长手机号注册或登录
-> 填写宝宝基础资料
-> 填写监护人和接送人
-> 填写健康、过敏、特殊照护
-> 园长确认班级和入园日期
-> 保健员审核健康风险
```

也允许园区代录：

```text
招生/园长代录宝宝基础资料
-> 系统生成邀请码
-> 家长注册或登录
-> 输入邀请码
-> 绑定宝宝和入托档案
-> 家长补充/确认资料
```


### 4.3 入托确认

参与角色：园长、保健员、家长。

工作流程：

```text
园长选择宝宝
-> 分配园区和班级
-> 设置入园日期和状态
-> 检查健康/过敏/特殊照护是否完整
-> 家长确认入托信息
-> 入托档案生效
```

生效后：

```text
老师/保育员可在班级工作台看到宝宝
家长可查看日报、考勤、照护、通知
园长可在运营统计看到在托人数
```


### 5.1 每日开园前

参与角色：园长、教师、保育员、保健员、安全后勤。

流程：

```text
园长查看今日工作台
-> 教师查看本班宝宝名单
-> 保育员查看照护任务
-> 保健员查看晨检准备
-> 安全后勤完成环境/消毒/设施检查
```

园长看：

- 今日预计到园人数。
- 员工到岗情况。
- 班级容量和异常提醒。
- 未关闭安全台账。
- 欠费或重要通知。


### 5.2 到园与晨检

参与角色：教师/保育员、保健员、家长。

流程：

```text
家长送宝宝到园
-> 老师/保育员记录到园
-> 保健员/老师记录晨检
-> 异常则生成健康观察或事故/异常记录
-> 家长收到必要提醒
```


### 5.3 一日照护

参与角色：保育员、教师、园长。

流程：

```text
保育员记录喂养/饮水/睡眠/如厕/体温/情绪
-> 教师补充活动和发展观察
-> 异常情况转健康观察或事故
-> 园长查看漏记和风险
```


### 5.4 教学与发展

参与角色：教师、园长、家长。

流程：

```text
教师创建教育计划/活动
-> 记录观察
-> 周期性发展评估
-> 园长查看班级教学完成度
-> 家长查看阶段反馈
```


### 5.5 家长日报

参与角色：教师、保育员、园长、家长、长辈。

流程：

```text
系统汇总考勤、照护、健康观察
-> 教师生成日报草稿
-> 教师编辑评语
-> 按策略直接发布或园长审核
-> 家长查看
-> 长辈以大字版查看摘要
```


### 5.6 离园与接送

参与角色：教师/保育员、家长、长辈/授权接送人、园长。

流程：

```text
家长维护接送人
-> 园区审核接送授权
-> 离园时核验接送人
-> 记录离园
-> 异常滞留或未授权接送触发预警
```


### 6.1 用药和过敏

参与角色：家长、保健员、教师/保育员。

流程：

```text
家长提交用药委托
-> 保健员审核
-> 教师/保育员执行用药
-> 记录实际用药
-> 家长查看结果
```


### 6.2 安全卫生台账

参与角色：安全后勤、保健员、园长。

流程：

```text
系统生成周期任务
-> 后勤/保健员填写台账
-> 逾期提醒
-> 园长查看未完成事项
-> 形成监管材料
```


### 6.3 传染病防控

参与角色：保健员、园长、家长。

流程：

```text
发现疑似症状
-> 保健员记录
-> 通知家长
-> 标记隔离/离园观察
-> 必要时上报
-> 复园审核
```


## 7. 财务运营流程

### 7.1 收费

参与角色：财务、园长、家长、老板。

流程：

```text
财务配置收费项目
-> 按宝宝/班级生成账单
-> 家长查看账单
-> 支付或线下确认
-> 财务标记已收/欠费/取消
-> 老板查看应收实收欠费
```


### 7.2 经营复盘

参与角色：老板、园长、财务/运营。

流程：

```text
统计招生线索
-> 统计试托转化
-> 统计托位使用率
-> 统计出勤和收费
-> 对比园区表现
-> 发现风险和改进点
```


## 8. 监管与合规流程

参与角色：园长、老板、保健员、安全后勤、平台管理员。

流程：

```text
日常业务记录沉淀
-> 自动形成监管字段
-> 园长补充缺失项
-> 保健/安全补齐台账
-> 导出监管材料
-> 老板查看各园区合规风险
```


## 9. 多园区统筹流程

参与角色：老板/机构管理员、园长、财务/运营、平台管理员。

核心原则：

```text
老板看所有园区
园长只管自己园区
教师/保育员只管授权班级
家长只看自己宝宝
平台管理员管系统，不替园区日常经营
```

老板每日看：

- 今日总在托人数。
- 各园区出勤率。
- 各园区托位使用率。
- 招生线索和试托转化。
- 收费应收、实收、欠费。
- 健康异常和安全事故。
- 未完成日报、未关闭台账。
- 园区排行榜和异常预警。

园长每日看：

- 本园区到离园。
- 班级照护完成度。
- 晨午检异常。
- 日报发布状态。
- 员工到岗和班级负荷。
- 本园区收费、招生、监管缺失。
