# UI 风格设计规范

最近更新：2026-07-19

## 1. 设计定位

本系统 UI 定位为“现代托育机构运营系统”，面向两类用户群体：

- **机构端**（园长、教师、保育员、财务、老板）：后台工作台风格，高效有序，信息密度适中，支持高频操作。
- **家长端**（父母、长辈）：温暖可信，简洁清晰，长辈模式提供大字高对比。

整体风格应体现：安全可信、专业有序、温和亲切。避免过度儿童化、营销落地页风格、大面积紫色/橙色渐变、过多装饰图形和过重阴影。

## 2. 目标风格

托育 + 保育系统应体现：

- 安全可信
- 专业有序
- 温和亲切
- 适合高频操作
- 信息密度适中
- 机构端清晰高效，家长端温暖可信

应避免：

- 过度儿童化
- 大面积紫色/橙色渐变
- 营销落地页风格
- 过多装饰图形
- 过重阴影
- 卡片套卡片

## 3. 推荐视觉方向

建议采用“现代托育机构运营系统”风格：

| 用途 | 建议色值 | 说明 |
|---|---|---|
| Primary | `#167D7F` | 稳定、健康、机构感 |
| Primary Hover | `#0F6B6D` | 主按钮 hover |

## 4. 页面级设计规范

### 登录/注册

- 登录页为托育机构系统入口，注册页与登录页保持统一视觉语言。
- 支持手机号+验证码注册/登录，微信登录预留入口。
- 提供忘记密码功能。
- 移动端登录页不溢出。

### 主布局

- 后台工作台结构，菜单按角色动态过滤（MainLayout 根据 userInfo.role 和 staffInfo.staffRole 动态显示）。
- Header 显示当前机构/班级上下文。
- 通知数量对接真实接口。

### Dashboard（今日工作台）

- 园长工作台（PrincipalWorkbench）：待办审核、风险预警和机构概况。
- 教师/保育员工作台：展示本班照护真实数据，支持照护任务标签页和漏记提醒。
- 老板驾驶舱（BossDashboard）：多园区聚合指标对比。
- 无数据时展示明确空状态，不展示假数据。

### 业务页面

- 核心业务页面统一使用中文骨架，不展示 demo/英文占位内容。
- 园长工作台、老板驾驶舱、保健工作台、财务工作台对接真实 API。
- 入托管理（OrganizationManagement）支持审核/转班/退托/暂停/复托操作。
- 家长申请（ParentApplications）支持请假/用药/接送提交。
- 系统管理后台（SystemManagement）支持 RBAC 角色/权限/菜单/用户管理。
- 保健工作台（HealthSafety）：聚合晨检、用药、过敏、食谱、台账、事故。
- 财务运营工作台（OperationsRegulatory + Billing）：含招生、收费、食谱、监管导出。
- 未完成能力显示“待接入”占位，不得假装可用。
- 使用通用页面组件（PageHeader、MetricCard、StatusTag、EmptyState），减少内联样式。

### 家长端

- AI 育儿、教育规划、家园协作、家庭管理：交互丰富，信息清晰。
- 家长日报：系统自动汇总考勤、照护、健康观察数据，教师编辑评语后发布。
- 长辈模式（ElderMode）：大字高对比 + 语音播报预留。
