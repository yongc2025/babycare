> 系统账号分为 `ADMIN`（机构管理端）和 `PARENT`（家长端）两大类，员工岗位通过员工管理模块区分，不同岗位拥有对应的工作台和操作权限。

## 角色对比总览

| 角色 | 系统账号 / 员工岗位 | 主要页面 | 核心操作 |
|---|---|---|---|
| 妈妈（未入园） | PARENT | AI育儿、教育规划、成长记录 | 注册、浏览、咨询 |
| 爷爷（已入园） | PARENT / ELDER | 长辈模式、家长日报、家庭管理 | 查看日报、接送、委托 |
| 家长（已入园） | PARENT | 家长日报、家园协作、家庭管理 | 查看宝宝动态、家园沟通、缴费 |
| 平台/系统管理员 | ADMIN | 系统管理 | 系统配置、账号权限、全局运维 |
| 机构负责人 | ADMIN | 今日工作台、运营监管、园所运营 | 全局管控、监管导出 |
| 园长 | ADMIN / DIRECTOR | 园所运营、班级照护、日报管理、健康安全 | 日常运营管理 |
| 教师 | ADMIN / TEACHER | 今日工作台、班级照护 | 考勤、晨检、照护记录 |
| 保育员 | ADMIN / CAREGIVER | 班级照护 | 喂养、午睡、如厕记录 |
| 保健员/保健医 | ADMIN / HEALTH_WORKER、HEALTH_DOCTOR | 健康安全 | 晨检、用药、传染病、食谱台账 |
| 安全/后勤负责人 | ADMIN / SAFETY_OFFICER、LOGISTICS_STAFF | 健康安全→安全台账 | 设施巡检、消毒、隐患排查 |
| 财务 | ADMIN / FINANCE | 运营监管→收费 | 账单、收费管理 |
| 运营/招生 | ADMIN / OPERATIONS_STAFF、ADMISSIONS_OFFICER | 运营监管→招生 | 线索跟进、报名审核、试托转化 |

## 1. 2 岁宝宝的妈妈（未入园）

### 角色画像
- 宝宝 2 岁，尚未入托
- 正在考察托育机构
- 关心育儿知识和早期教育规划
- 需要通过平台了解机构、建立信任

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[打开好芽儿登录页]
    B --> C{已有账号？}
    C -->|否| D[点击"注册"]
    C -->|是| E[输入账号密码登录]
    
    D --> F[填写注册信息\n用户名/密码/昵称]
    F --> G[同意用户协议]
    G --> H[注册成功]
    H --> E
    
    E --> I[首次进入系统\n角色：PARENT]
    I --> J[浏览系统功能]
    
    J --> K[查看「AI育儿」\n了解智能育儿助手]
    J --> L[查看「教育规划」\n浏览课程体系]
    J --> M[查看「成长记录」\n了解记录方式]
    J --> N[查看「家园协作」\n了解家校互动]
    
    K --> O[产生兴趣]
    L --> O
    M --> O
    N --> O
    
    O --> P{决定咨询？}
    P -->|是| Q[通过机构联系方式咨询\n或等待后续招生功能]
    P -->|否| R[继续浏览 / 退出]
    
    Q --> S[结束]
    R --> S
```

### 日常使用流程图

```mermaid
flowchart TD
    A[日常登录] --> B[登录 PARENT 账号]
    B --> C{今天做什么？}
    
    C -->|学习育儿知识| D[打开「AI育儿」]
    C -->|规划早教| E[打开「教育规划」]
    C -->|记录宝宝成长| F[打开「成长记录」\n拍照/写日记]
    C -->|了解机构动态| G[打开「家园协作」\n查看公告/活动]
    C -->|查看家庭动态| H[打开「家庭管理」\n管理家庭成员]
    
    D --> I[向 AI 提问育儿问题\n获取建议]
    E --> J[查看/选择教育课程\n了解发展目标]
    F --> K[上传照片/视频\n记录宝宝里程碑]
    G --> L[查看机构通知\n参与话题讨论]
    H --> M[邀请家人加入\n管理宝宝信息]
    
    I --> N[结束]
    J --> N
    K --> N
    L --> N
    M --> N
```

## 2. 2 岁宝宝的爷爷（已入园）

### 角色画像
- 宝宝已在托育机构
- 主要关注宝宝每日在园情况
- 可能需要负责接送
- 偏好简单大字界面（长辈模式）
- 不太熟悉智能手机操作

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[收到家庭邀请\n（妈妈发送邀请码）]
    B --> C{已有账号？}
    
    C -->|否| D[打开注册页]
    D --> E[填写信息注册\n建议使用大字体]
    E --> F[注册成功]
    F --> G[登录系统]
    
    C -->|是| G
    
    G --> H[进入「家庭管理」]
    H --> I[输入邀请码\n加入家庭]
    I --> J[确认加入成功\n可看到宝宝信息]
    
    J --> K{需要大字模式？}
    K -->|是| L[进入「长辈模式」]
    K -->|否| M[使用标准模式]
    
    L --> N[查看宝宝今日状态\n熟悉界面布局]
    M --> N
    
    N --> O[查看「家长日报」\n了解日报内容和位置]
    O --> P[查看「接送管理」\n了解接送人设置]
    P --> Q[结束首次引导]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[日常打开] --> B[打开好芽儿]
    B --> C{使用长辈模式？}
    C -->|是| D[进入「长辈模式」\n大字体、简版界面]
    C -->|否| E[标准模式]
    
    D --> F[首页概览：\n今日宝宝状态]
    E --> F
    
    F --> G[查看考勤状态\n已到园/已接走]
    G --> H[查看「家长日报」\n阅读老师评语/今日 summary]
    H --> I[查看照护摘要\n吃了什么/睡了多久]
    
    I --> J{今天要接送？}
    J -->|是| K[查看/授权接送人]
    J -->|否| L[浏览宝宝成长记录\n或退出]
    
    K --> M[确认接送安排]
    M --> L
```

## 3. 机构负责人

### 角色画像
- 投资人或创办人
- 关注全局运营数据
- 不参与日常班级管理
- 需要监管报表和导出能力

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[联系技术支持\n创建 ADMIN 账号]
    B --> C[首次登录]
    C --> D[进入「园所运营」]
    
    D --> E[创建机构信息\n名称/地址/电话/许可证]
    E --> F[完善机构资质\n备案编号/等级/法人]
    F --> G[设置班级\n豆芽一班 等]
    G --> H[添加管理层员工\n园长/财务负责人]
    
    H --> I[进入「今日工作台」\n查看全局仪表盘]
    I --> J[熟悉各模块：\n班级照护 / 日报管理\n健康安全 / 运营监管]
    
    J --> K[进入「运营监管」\n查看监管导出字段]
    K --> L[确认 MISSING 字段\n补充机构信息]
    L --> M[结束初始化]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[日常登录] --> B[登录 ADMIN 账号]
    B --> C[进入「今日工作台」\n查看全局概览]
    
    C --> D[检查关键指标：\n出勤率 / 在托人数\n健康异常 / 待办事项]
    
    D --> E{需要深入？}
    
    E -->|查看运营数据| F[进入「运营监管」]
    E -->|检查安全隐患| G[进入「健康安全」]
    E -->|审阅日报| H[进入「日报管理」\n抽查已发布日报]
    E -->|了解家长反馈| I[进入「家园协作」\n查看互动情况]
    
    F --> J[查看监管报表\n导出数据用于上报]
    G --> K[查看事故/台账\n确保安全合规]
    H --> L[确认日报质量\n给出指导意见]
    I --> M[查看家长留言\n了解满意度]
    
    J --> N[结束巡园]
    K --> N
    L --> N
    M --> N
```

## 4. 园长

### 角色画像
- 机构日常运营管理者
- 管理教师团队和班级
- 审核日报、处理异常
- 规划教育课程和活动

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[机构负责人创建账号\n或获得 ADMIN 权限]
    B --> C[首次登录]
    C --> D[进入「园所运营」]
    
    D --> E[查看/编辑机构信息]
    E --> F[创建/管理班级\n设置月龄范围/容量]
    F --> G[添加员工\n招聘教师/保育员/财务]
    G --> H[设置员工信息\n姓名/手机/岗位/状态]
    
    H --> I[进入「班级照护」\n验证班级数据]
    I --> J[进入「健康安全」\n了解安全模块布局]
    J --> K[进入「教育规划」\n查看/设置课程体系]
    K --> L[结束首次设置]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[每日工作] --> B[登录 ADMIN 账号]
    B --> C[「今日工作台」\n查看今日概览]
    
    C --> D[检查考勤情况\n哪些宝宝未到园]
    D --> E[进入「班级照护」\n巡查各班级]
    
    E --> F[查看今日健康观察\n是否有异常记录]
    F --> G[查看照护记录\n喂养/睡眠是否正常]
    
    G --> H{需要处理事项？}
    
    H -->|审核日报| I[进入「日报管理」\n审阅待发布日报\n确认后发布]
    H -->|处理安全事件| J[进入「健康安全」\n处理用药委托/事故\n关闭台账]
    H -->|与家长沟通| K[进入「家园协作」\n回复家长留言\n发布公告]
    H -->|调整课程| L[进入「教育规划」\n更新活动安排]
    
    I --> M[结束]
    J --> M
    K --> M
    L --> M
```

## 5. 教师

### 角色画像
- 直接带班的一线老师
- 负责晨检、照护记录、日报
- 与家长直接沟通
- 需要高效记录工具

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[园长在员工管理中添加\n手机号/岗位=老师]
    B --> C[收到邀请/账号信息\n密码初始设置]
    C --> D[首次登录 ADMIN 账号]
    
    D --> E[进入「班级照护」\n查看分配班级]
    E --> F[查看今日幼儿列表\n熟悉系统界面]
    F --> G[进入「园所运营」\n查看班级和幼儿信息]
    G --> H[进入「今日工作台」\n了解工作台布局]
    
    H --> I[模拟操作：\n找到"小芽芽"的考勤/照护入口]
    I --> J[了解如何记录\n健康观察和照护记录]
    J --> K[了解日报生成流程\n知道草稿→编辑→发布]
    K --> L[结束培训]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[早晨到园] --> B[登录 ADMIN 账号]
    B --> C[进入「班级照护」\n选择今日班级]
    
    C --> D[晨间接待\n逐一点名签到]
    D --> E[执行晨检\n体温/一摸二看三问四查\n记录异常情况]
    
    E --> F[上午活动\n记录照护：\n- 饮水/加餐\n- 如厕情况]
    F --> G[午餐时间\n记录照护：\n- 进食量/菜品\n- 过敏情况]
    
    G --> H[午睡时间\n记录照护：\n- 入睡时间/时长\n- 睡眠质量]
    H --> I[下午活动\n记录照护：\n- 午点/活动\n- 体温复检]
    
    I --> J[放学前\n进入「日报管理」\n生成日报草稿]
    J --> K[编辑日报\n补充老师评语\n确认数据完整]
    K --> L[提交日报给园长审核\n或直接发布]
    L --> M[结束一日工作]
```

## 6. 保育员

### 角色画像
- 负责宝宝日常生活照料
- 关注喂养、睡眠、如厕、卫生
- 与教师配合工作
- 偏重记录而非管理

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[园长在员工管理中添加\n手机号/岗位=保育员]
    B --> C[首次登录 ADMIN 账号]
    C --> D[进入「班级照护」\n查看负责班级]
    
    D --> E[查看今日幼儿名单\n了解每位宝宝]
    E --> F[了解照护记录类型\n喂养/睡眠/如厕/体温]
    F --> G[了解健康观察入口\n体温/症状记录]
    
    G --> H[尝试新建一条\n照护记录（模拟）]
    H --> I[确认记录保存成功\n可在列表查看]
    I --> J[结束]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[日常工作] --> B[登录 ADMIN 账号]
    B --> C[进入「班级照护」]
    
    C --> D[早餐/午餐时间\n记录照护：\n- 喂养记录 FEEDING\n- 食量/菜品\n- 过敏反应]
    
    D --> E[上午/下午点心\n记录照护：\n- 饮水 WATER\n- 进食量]
    
    E --> F[午睡时间\n记录照护：\n- 睡眠 SLEEP\n- 入睡/醒来时间\n- 睡眠质量]
    
    F --> G[如厕/换尿布\n记录照护：\n- 如厕 TOILET\n- 正常/异常]
    
    G --> H[体温测量\n记录照护：\n- 体温 TEMPERATURE\n- 异常上报老师]
    
    H --> I[发现异常？]
    I -->|是| J[通知班级老师\n在健康观察中记录]
    I -->|否| K[继续日常工作]
    
    J --> K
```

### 日常使用流程图

```mermaid
flowchart TD

    I --> L[结束]
    J --> L
    K --> L
```

## 8. 保健员/保健医

### 角色画像
- 负责园区卫生保健、健康异常处理
- 管理用药委托、过敏、传染病防控
- 负责食谱、营养分析、食品留样
- 输出卫生保健台账和监管材料

### 首次使用流程图

```mermaid
flowchart TD
    A[开始] --> B[园长在员工管理中\n添加保健员/保健医]
    B --> C[首次登录 ADMIN 账号]
    C --> D[进入「健康安全」]
    
    D --> E[查看晨检模块\n了解晨检流程]
    E --> F[查看用药管理\n了解用药委托审核流程]
    F --> G[查看过敏记录\n确认已知过敏宝宝]
    
    G --> H[查看传染病防控模块\n了解上报/隔离/复园流程]
    H --> I[查看食谱管理\n了解营养分析入口]
    I --> J[结束初始化]
```

### 日常使用流程图

```mermaid
flowchart TD
    A[每日工作] --> B[登录 ADMIN 账号]
    B --> C[进入「健康安全」]
    
    C --> D[查看晨午检异常\n处理健康观察记录]
    D --> E{有用药委托？}
    E -->|是| F[审核用药委托\n确认用药方案]
    E -->|否| G[继续日常]
    
    F --> G
    G --> H[检查过敏宝宝\n今日食谱是否安全]
```
