# T040 真实数据完整试跑测试方案

## 1. 测试目标

本测试用于验证“好芽儿”当前版本是否已经从页面骨架进入真实托育保育业务闭环。

核心目标：

- 使用真实前后端服务和本地 MySQL 数据，不使用 mock 数据。
- 跑通一条最小主链：机构、班级、员工、入托、考勤、照护、日报、家长查看。
- 记录所有阻断问题、接口问题、字段问题、权限问题、交互问题和样式问题。
- 为后续 T041+ 修复任务提供明确输入。

本轮不追求全量功能通过，优先判断：

```text
系统是否已经具备可演示的真实业务路径
```

## 2. 测试执行原则

- 必须先阅读：
  - `docs/AGENT_WORKFLOW.md`
  - `docs/PROJECT_STATUS.md`
  - `docs/DEVELOPMENT_TASKS.md`
  - 本文件
- 如发现乱码，必须先用 `node`/`rg` 验证真实 UTF-8 内容。
- 不允许修改业务代码来绕过问题。
- 不允许新增 mock fallback 或手工伪造前端数据。
- 如需要准备数据库数据，必须记录 SQL 或操作步骤。
- 每个失败点必须记录复现路径、期望结果、实际结果和截图/日志位置。

## 7. L1 主链用例

### TC-L1-001 登录和角色菜单

前置条件：

- 前后端已启动。
- 有可登录账号。

步骤：

1. 打开 `http://localhost:3001/login`。
2. 输入账号密码登录。
3. 进入系统首页。
4. 查看左侧菜单。

期望结果：

- 登录成功后进入工作台。
- ADMIN 可见园所运营、班级照护、日报管理、健康安全、运营监管等菜单。
- PARENT 可见成长记录、AI 育儿、教育规划、家园协作、家长日报、长辈模式、家庭管理等菜单。
- 不同角色菜单边界清晰。

记录重点：

- 如果登录成功但角色不对，记录当前用户信息和菜单截图。
- 如果无法进入后台，记录接口响应和后端日志。

### TC-L1-002 创建或选择机构

路径：

```text
左侧菜单 -> 园所运营
```

步骤：

1. 进入园所运营页面。
2. 查看是否已有机构。
3. 如无机构，创建“好芽儿 T040 测试园”。
4. 保存后刷新页面。

期望结果：

- 机构创建成功。
- 刷新后机构仍存在。
- 机构选择控件或列表能显示新机构。

失败记录：

- 创建按钮不可用。
- 表单字段缺失。
- 接口 401/403/500。
- 创建成功但列表不刷新。

### TC-L1-003 创建班级

步骤：

1. 在园所运营页面选择测试机构。
2. 创建“豆芽一班”。
3. 填写月龄范围和容量。
4. 保存后查看班级列表。

期望结果：

- 班级创建成功。
- 班级归属于当前机构。
- 班级状态为启用或可用。

### TC-L1-004 添加员工

步骤：

1. 在园所运营页面进入员工区域。
2. 添加“李老师”。
3. 填写手机号和岗位信息。
4. 保存后查看员工列表。

期望结果：

- 员工创建成功。
- 员工归属于当前机构。
- 员工列表显示姓名、手机号、角色/岗位。

### TC-L1-005 创建入托档案

步骤：

1. 在园所运营页面进入入托档案区域。
2. 选择“豆芽一班”。
3. 新增宝宝“小芽芽”入托档案。
4. 填写生日、入托日期、监护人信息等必填字段。
5. 保存后查看班级入托档案列表。

期望结果：

- 入托档案创建成功。
- 宝宝出现在当前班级档案列表。
- 入托状态为在托或有效状态。

重点观察：

- 表单是否要求家庭/宝宝先存在。
- 入托档案是否能直接从机构端创建。
- 如果必须绑定家长账号，记录为产品流程缺口。

### TC-L1-006 今日考勤

路径：

```text
左侧菜单 -> 班级照护
```

步骤：

1. 选择测试机构。
2. 选择“豆芽一班”。
3. 找到“小芽芽”。
4. 执行到园操作。
5. 刷新页面。

期望结果：

- 今日考勤记录创建成功。
- 宝宝状态变为已到园。
- 刷新后状态保持。
- 首页/工作台出勤统计能反映变化。

### TC-L1-007 晨午检或健康观察

步骤：

1. 在班级照护页选择“小芽芽”。
2. 新增晨检或健康观察。
3. 填写体温、精神状态、异常标记等字段。
4. 保存后查看记录。

期望结果：

- 健康观察保存成功。
- 记录出现在今日班级照护页或健康安全页。

### TC-L1-008 一日照护记录

步骤：

1. 在班级照护页选择“小芽芽”。
2. 新增照护记录。
3. 至少填写一条进餐、午睡、如厕或活动记录。
4. 保存后刷新页面。

期望结果：

- 照护记录保存成功。
- 刷新后记录仍存在。
- 后续日报草稿能读取该记录。

### TC-L1-009 生成日报草稿

路径：

```text
左侧菜单 -> 日报管理
```

步骤：

1. 选择测试机构。
2. 选择“豆芽一班”。
3. 选择“小芽芽”和今天日期。
4. 点击生成日报草稿。
5. 检查草稿内容是否汇总今日考勤、健康观察和照护记录。

期望结果：

- 日报草稿生成成功。
- 草稿内容包含真实记录。
- 无记录字段明确展示暂无记录，不使用假内容。

### TC-L1-010 编辑并发布日报

步骤：

1. 在日报管理页编辑草稿字段。
2. 保存草稿。
3. 发布日报。
4. 刷新页面。

期望结果：

- 保存成功。
- 发布成功。
- 日报状态变为已发布。
- 刷新后状态保持。

### TC-L1-011 家长端查看日报

路径：

```text
左侧菜单 -> 家长日报
```

步骤：

1. 使用 PARENT 账号登录，或在当前账号有权限时进入家长日报页面。
2. 选择对应宝宝和日期。
3. 查看已发布日报。

期望结果：

- 家长端只展示已发布日报。
- 能看到考勤、照护、健康观察等日报内容。
- 未绑定宝宝或无权限时显示明确空状态/错误状态。

重点记录：

- 如果当前系统无法把家长账号和入托宝宝绑定，记录为主链阻断问题。

## 11. 接口与日志检查

测试过程中如出现问题，必须记录：

- 浏览器 Network：
  - 请求 URL
  - HTTP 状态码
  - Request Payload
  - Response Body
- 后端日志：
  - `logs/backend.log` 对应时间段错误
- 前端日志：
  - `logs/frontend.log`
  - 浏览器 Console 错误

常见问题分类：

| 分类 | 说明 |
|---|---|
| AUTH | 登录、Token、角色、权限 |
| API | 接口 404/500/字段不匹配 |
| DATA | 数据创建成功但下游查不到 |
| UX | 页面流程不清楚、按钮不可用、提示缺失 |
| UI | 样式错位、颜色不清楚、移动端溢出 |
| PRODUCT | 产品流程断点，如家长无法绑定入托宝宝 |
