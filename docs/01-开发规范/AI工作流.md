# AI 工作流细则

本文定义具体的开发执行流程、任务选择规则、交付要求和验证命令。
它是 `AGENTS.md` 和 `.github/copilot-instructions.md` 所路由的工作流细则，本身不是独立入口。

目标：确保长期开发即使中断，也能按固定流程恢复，避免遗忘上下文、重复造轮子、超出产品边界或继续堆叠 demo 功能。

## 1. 固定工作流程

本文是工作流细则文档，由入口文件（`AGENTS.md` / `.github/copilot-instructions.md`）路由后读取。

按以下流程执行，并按任务范围补读相关文档：

- 当前状态、任务选择、依赖关系：`docs/03-项目管理/项目状态.md`、`docs/03-项目管理/开发任务.md`
- 数据库、Entity、字段、枚举、索引、关系：`docs/02-架构设计/数据模型.md`
- Controller 路由、前端 API、Service 公共方法、Store action：`docs/02-架构设计/API设计.md`
- 架构边界、微服务策略、跨域数据流、硬件和监管策略：`docs/02-架构设计/架构决策.md`
- 产品方向和需求取舍：`docs/04-产品需求/托育保育产品规划.md`、`docs/04-产品需求/需求差异与采纳矩阵.md`

不要为每个小问题完整读取所有长文档；先读入口文件路由后，再根据当前任务读取相关文档和代码。

## 2. 编码优先规则

编码规范（乱码处理、UTF-8 编码等）以 `docs/01-开发规范/编码规范.md` §1 为唯一真源，本文不再重复。

## 3. 固定执行流程

任何开发任务都必须按以下流程执行：

```text
读取入口文档
-> 查看任务清单
-> 确认当前任务
-> 扫描乱码和构建状态
-> 批量读取相关代码和引用
-> 做最小设计并判断复用已有/扩展现有/新建
-> 实现后端
-> 实现前端
-> 批量验证
-> 更新文档
-> 输出本轮总结
-> 进入下一任务
```

执行细则以 `docs/01-开发规范/编码规范.md` 的“抽象设计与开发效率”为准：开发前批量读取，改动时批量编辑，完成一整层或一个业务闭环后批量验证；抽象优先服务真实业务边界和稳定复用点。

## 4. 当前任务选择规则

1. 优先查找 `docs/03-项目管理/开发任务.md` 中状态为 `In Progress` 的任务。
2. 如果存在 `In Progress`，继续该任务。
3. 如果没有 `In Progress`，选择优先级最高的 `Pending` 任务。
4. 如果任务依赖未完成，不得跳过依赖直接开发。
5. 如果任务范围不清，先补充任务说明和验收标准。

## 5. 开发边界

必须围绕托育 + 保育产品主链推进：

```text
机构 -> 班级 -> 宝宝 -> 考勤 -> 保育记录 -> 日报 -> 家长同步
```

短期不优先做：

- 非核心 AI 包装
- 纯展示型 Dashboard
- 泛社区功能
- 与托育主链无关的大型重构
- 只增加页面但不接真实 API 的功能

## 6. 禁止事项

> 以下规则以 `docs/01-开发规范/编码规范.md` 为唯一真源，本文只列出与工作流执行直接相关的禁止项：

- 不得绕过现有 `request`/API service 自行散落写请求。
- 不得把复杂业务逻辑堆进 React 页面组件。
- 不得在 Controller 中写复杂业务逻辑。
- 不得新增未记录的数据表、接口或核心方法。
- 不得引入新框架或大依赖，除非先记录原因并确认必要性。
- 不得改动无关文件制造大面积 diff。

Mock/假数据、复用/扩展/新建等规则详见 `编码规范.md`。

## 7. 每轮交付要求

### 任务状态与文档关联（不可断）

创建或更新任务时，以下三维度必须同步更新，保持关联不可断裂：

| 维度 | 负责文档 | 说明 |
|---|---|---|
| 任务状态 | `docs/03-项目管理/开发任务.md` | 状态、依赖、验收标准，逐条记录 |
| 项目总览 | `docs/03-项目管理/项目状态.md` | 整体进度、已完成能力、风险、下一步 |
| 文档更新 | 见下方清单 | 按变更类型更新对应专项文档 |

### 必更文档清单

每完成一个任务，**必须**更新：

- `docs/03-项目管理/项目状态.md` — 总览进度
- `docs/03-项目管理/开发任务.md` — 任务状态和验收
- `docs/02-架构设计/数据模型.md`，如果涉及实体、表、字段
- `docs/02-架构设计/API设计.md`，如果涉及接口
- `docs/01-开发规范/变更日志.md` — 重要变更记录

> 任务状态和文档关联不可断：不得只改任务状态而不同步更新项目总览和相关文档。

### 最终回复模板

每次交付的回复必须包含：

- 本轮完成内容
- 修改文件清单
- 验证结果
- 未验证原因或遗留风险
- 下一步建议

## 8. 验证命令

前端：

```bash
cd frontend
npm run build
```

后端：

```bash
cd backend
mvn test
```

如果 Maven 不可用，至少记录：

```text
后端未验证：当前环境缺少 mvn 命令或依赖不可用。
```

## 10. 长期记忆原则

聊天记录不是长期记忆来源。长期记忆只来自仓库内文档。

如果文档和代码冲突：

1. 先读代码确认事实。
2. 更新文档。
3. 再继续开发。
