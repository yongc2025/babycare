## 1. 测试目标

验证好芽儿托育保育平台是否具备产品交付条件，覆盖：

1. **冒烟测试**：系统能否正常启动和登录
2. **功能联调测试**：各模块 API 能否正确协同工作
3. **验收测试**：核心业务流程能否从用户视角跑通
4. **安全测试**：认证、权限、输入安全防护是否到位
5. **性能测试**：关键 API 响应时间是否在可接受范围
6. **边界与异常测试**：系统对异常输入的容错能力
7. **全角色测试**：11 种角色（系统管理员、机构负责人、园长、教师、保育员、保健员、安全后勤、财务、运营招生、家长、长辈）的完整功能流转

## 6. 缺陷管理

测试过程中发现的缺陷按以下级别分类：

| 级别 | 定义 | 处理要求 |
|---|---|---|
| **Critical** | 系统无法启动、核心功能不可用、数据丢失 | 必须立即修复 |
| **Major** | 重要功能异常、业务流程中断 | 本轮必须修复 |
| **Minor** | 非核心功能异常、UI 显示问题 | 可延期修复 |
| **Trivial** | 文案错误、样式微调 | 记录即可 |
