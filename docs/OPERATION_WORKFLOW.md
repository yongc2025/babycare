# 系统运转工作流程

最近更新：2026-07-16

## 1. 总体定位

本系统不是单一“家长记录工具”，而是托育保育机构从开通、建园、招生、入托、照护、家园共育、健康安全、收费、监管到经营复盘的协同系统。

完整运转链路：

```text
平台开通
-> 机构/集团入驻
-> 多园区搭建
-> 园区初始化
-> 招生报名
-> 宝宝入托
-> 日常照护和教学
-> 家长同步和协作
-> 健康安全和保健
-> 财务收费
-> 监管和经营分析
-> 持续优化
```

## 2. 组织层级

推荐组织模型：

```text
平台 Platform
└── 机构主体 / 集团 Tenant
    ├── 园区 Campus / Organization A
    │   ├── 班级 Classroom
    │   ├── 员工 Staff
    │   ├── 入托档案 Enrollment
    │   └── 日常业务记录
    └── 园区 Campus / Organization B
        ├── 班级 Classroom
        ├── 员工 Staff
        ├── 入托档案 Enrollment
        └── 日常业务记录
```

当前系统现状：

- 已有 `Organization`，目前更接近单个园区。
- 还没有独立的机构主体/集团模型。
- 多园区统筹需要后续通过 T051-T053 建立。

## 3. 新上线完整流程

### 3.1 平台初始化

参与角色：平台/系统管理员。

工作流程：

```text
配置系统基础参数
-> 创建平台管理员
-> 配置角色权限
-> 配置数据字典
-> 配置短信/文件/通知/登录策略
-> 开启机构入驻
```

需要完成的功能：

- 系统设置。
- 平台账号管理。
- RBAC 权限配置。
- 数据字典。
- 审计日志。
- 系统健康检查。

当前系统已有：

- 开发环境管理员账号初始化。
- 配置文件支持数据库、CORS、上传和 JWT。

当前缺口：

- 缺少正式系统管理后台。
- 缺少 RBAC。
- 缺少数据字典和审计日志。

关联任务：T048、T049、T050。

### 3.2 机构/集团入驻

参与角色：平台管理员、老板/机构管理员。

工作流程：

```text
平台管理员创建机构主体
-> 创建老板/机构管理员账号
-> 老板登录
-> 补充机构主体资料
-> 创建一个或多个园区
```

老板需要管理：

- 机构主体名称、营业资质、联系人。
- 多个园区。
- 每个园区负责人。
- 多园区经营数据。

当前系统已有：

- 可创建 `Organization`。
- 可维护单个机构/园区的基础资料和监管字段。

当前缺口：

- 缺少机构主体/集团。
- 缺少老板角色。
- 缺少多园区列表和园区负责人任命。

关联任务：T051、T052、T053。

### 3.3 多园区搭建

参与角色：老板/机构管理员、园长。

工作流程：

```text
老板创建园区
-> 分配园长
-> 园长接管园区
-> 园长补充园区资料
-> 创建班级
-> 配置托位和月龄范围
-> 创建/邀请员工
-> 分配员工岗位和班级
```

老板关注：

- 哪些园区已开通。
- 哪个园区谁负责。
- 各园区托位、员工、招生和风险情况。

园长关注：

- 本园区资料是否完整。
- 班级是否创建。
- 员工是否到位。
- 班级容量和师生比是否合理。

当前系统已有：

- `/organization-management` 可创建机构、班级、员工和入托档案。
- `Staff` 有园长、老师、保育员、财务岗位。

当前缺口：

- 园区负责人任命。
- 班级员工分配。
- 员工岗位不足：缺少保健员、安全后勤、运营招生。
- 园长数据范围未和园区绑定。

关联任务：T053、T054、T055、T061、T065、T067。

## 4. 招生到入托流程

### 4.1 招生报名

参与角色：招生/运营、家长、园长。

工作流程：

```text
家长咨询或扫码报名
-> 招生/运营创建线索
-> 跟进沟通
-> 安排参观/试托
-> 园长审核托位和班级
-> 确认是否转入托
```

当前系统已有：

- `AdmissionLead` 招生线索。
- 试托基础字段和状态。
- `/operations-regulatory` 可查看招生相关数据。

当前缺口：

- 招生/运营角色。
- 招生漏斗。
- 跟进记录。
- 报名表。
- 线索转入托。

关联任务：T067、T069、T075。

### 4.2 宝宝资料创建

参与角色：家长、招生/运营、园长、保健员。

工作流程：

```text
家长手机号注册或登录
-> 填写宝宝基础资料
-> 填写监护人和接送人
-> 填写健康、过敏、特殊照护
-> 园长确认班级和入园日期
-> 保健员审核健康风险
```

也允许园区代录：

```text
招生/园长代录宝宝基础资料
-> 系统生成邀请码
-> 家长注册或登录
-> 输入邀请码
-> 绑定宝宝和入托档案
-> 家长补充/确认资料
```

当前系统已有：

- 家庭、宝宝、入托档案。
- 入托档案有过敏、健康备注、紧急联系人。

当前缺口：

- 家长手机号注册与短信验证。
- 邀请码绑定宝宝和入托档案。
- 家长补资料。
- 保健员审核。

关联任务：T070、T076、T078、T079、T080。

### 4.3 入托确认

参与角色：园长、保健员、家长。

工作流程：

```text
园长选择宝宝
-> 分配园区和班级
-> 设置入园日期和状态
-> 检查健康/过敏/特殊照护是否完整
-> 家长确认入托信息
-> 入托档案生效
```

生效后：

```text
老师/保育员可在班级工作台看到宝宝
家长可查看日报、考勤、照护、通知
园长可在运营统计看到在托人数
```

当前系统已有：

- `Enrollment` 入托档案。
- 创建后默认 `ACTIVE`。
- 班级照护和日报可基于入托档案运行。

当前缺口：

- 入托审核状态。
- 家长确认。
- 健康审核。
- 转班、退托、复托。

关联任务：T056、T074、T077。

## 5. 日常园区运转流程

### 5.1 每日开园前

参与角色：园长、教师、保育员、保健员、安全后勤。

流程：

```text
园长查看今日工作台
-> 教师查看本班宝宝名单
-> 保育员查看照护任务
-> 保健员查看晨检准备
-> 安全后勤完成环境/消毒/设施检查
```

园长看：

- 今日预计到园人数。
- 员工到岗情况。
- 班级容量和异常提醒。
- 未关闭安全台账。
- 欠费或重要通知。

当前系统已有：

- Dashboard、TeacherWorkbench、HealthSafety、OperationsRegulatory。

当前缺口：

- 园长专属待办。
- 员工排班/到岗。
- 后勤周期任务。

关联任务：T054、T059、T066。

### 5.2 到园与晨检

参与角色：教师/保育员、保健员、家长。

流程：

```text
家长送宝宝到园
-> 老师/保育员记录到园
-> 保健员/老师记录晨检
-> 异常则生成健康观察或事故/异常记录
-> 家长收到必要提醒
```

当前系统已有：

- 考勤到园、缺勤。
- 健康观察。
- 异常/事故。

当前缺口：

- 接送扫码/签名。
- 晨检异常自动通知。
- 到离园硬件事件转业务记录。

关联任务：T057、T062、T025 后续增强。

### 5.3 一日照护

参与角色：保育员、教师、园长。

流程：

```text
保育员记录喂养/饮水/睡眠/如厕/体温/情绪
-> 教师补充活动和发展观察
-> 异常情况转健康观察或事故
-> 园长查看漏记和风险
```

当前系统已有：

- `CareRecord`。
- `TeacherWorkbench` 可新增照护记录。

当前缺口：

- 保育员专属任务清单。
- 漏记提醒。
- 补录审核。
- 本班授权。

关联任务：T057、T059、T060。

### 5.4 教学与发展

参与角色：教师、园长、家长。

流程：

```text
教师创建教育计划/活动
-> 记录观察
-> 周期性发展评估
-> 园长查看班级教学完成度
-> 家长查看阶段反馈
```

当前系统已有：

- 教育规划。
- 成长评估。
- 家长可查看部分成长内容。

当前缺口：

- 教师端教学工作台。
- 园长教学质量统计。
- 家长端发展报告。

关联任务：T057 后续扩展。

### 5.5 家长日报

参与角色：教师、保育员、园长、家长、长辈。

流程：

```text
系统汇总考勤、照护、健康观察
-> 教师生成日报草稿
-> 教师编辑评语
-> 按策略直接发布或园长审核
-> 家长查看
-> 长辈以大字版查看摘要
```

当前系统已有：

- 日报生成、AI 草稿、编辑、发布。
- 家长日报。
- 长辈模式。

当前缺口：

- 发布审核策略。
- 已读回执。
- 长辈授权。

关联任务：T058、T073。

### 5.6 离园与接送

参与角色：教师/保育员、家长、长辈/授权接送人、园长。

流程：

```text
家长维护接送人
-> 园区审核接送授权
-> 离园时核验接送人
-> 记录离园
-> 异常滞留或未授权接送触发预警
```

当前系统已有：

- 接送人和委托接送模型。
- 家长端可查看接送信息。

当前缺口：

- 接送审核页面。
- 离园核验。
- 滞留预警。
- 接送记录不可篡改策略。

关联任务：T071、T073 后续扩展。

## 6. 健康安全运转流程

### 6.1 用药和过敏

参与角色：家长、保健员、教师/保育员。

流程：

```text
家长提交用药委托
-> 保健员审核
-> 教师/保育员执行用药
-> 记录实际用药
-> 家长查看结果
```

当前系统已有：

- 用药委托、审核、用药执行模型。

当前缺口：

- 家长提交页面。
- 保健员审核工作台。
- 用药提醒。

关联任务：T071、T062。

### 6.2 安全卫生台账

参与角色：安全后勤、保健员、园长。

流程：

```text
系统生成周期任务
-> 后勤/保健员填写台账
-> 逾期提醒
-> 园长查看未完成事项
-> 形成监管材料
```

当前系统已有：

- 安全卫生台账。
- 健康安全页面可新增台账。

当前缺口：

- 周期任务。
- 逾期提醒。
- 责任人和整改闭环。

关联任务：T066。

### 6.3 传染病防控

参与角色：保健员、园长、家长。

流程：

```text
发现疑似症状
-> 保健员记录
-> 通知家长
-> 标记隔离/离园观察
-> 必要时上报
-> 复园审核
```

当前系统已有：

- 健康观察和异常记录。

当前缺口：

- 传染病专用流程。
- 复园证明。
- 班级风险提醒。

关联任务：T063。

## 7. 财务运营流程

### 7.1 收费

参与角色：财务、园长、家长、老板。

流程：

```text
财务配置收费项目
-> 按宝宝/班级生成账单
-> 家长查看账单
-> 支付或线下确认
-> 财务标记已收/欠费/取消
-> 老板查看应收实收欠费
```

当前系统已有：

- 收费项目和账单模型。
- 运营监管页可聚合部分收费信息。

当前缺口：

- 家长账单页。
- 支付接口。
- 退费、优惠、发票。
- 财务专属工作台。

关联任务：T068、T072。

### 7.2 经营复盘

参与角色：老板、园长、财务/运营。

流程：

```text
统计招生线索
-> 统计试托转化
-> 统计托位使用率
-> 统计出勤和收费
-> 对比园区表现
-> 发现风险和改进点
```

当前系统已有：

- 园长驾驶舱。
- 运营监管页。

当前缺口：

- 多园区经营对比。
- 招生漏斗。
- 趋势报表。

关联任务：T052、T069。


当前系统已有：

- 监管导出行。
- 运营监管页。

当前缺口：

- 等级评定。
- 多园区合规风险。

关联任务：T052、T062、T066 后续扩展。

## 9. 多园区统筹流程

参与角色：老板/机构管理员、园长、财务/运营、平台管理员。

核心原则：

```text
老板看所有园区
园长只管自己园区
教师/保育员只管授权班级
家长只看自己宝宝
平台管理员管系统，不替园区日常经营
```

老板每日看：

- 今日总在托人数。
- 各园区出勤率。
- 各园区托位使用率。
- 招生线索和试托转化。
- 收费应收、实收、欠费。
- 健康异常和安全事故。
- 未完成日报、未关闭台账。
- 园区排行榜和异常预警。

园长每日看：

- 本园区到离园。
- 班级照护完成度。
- 晨午检异常。
- 日报发布状态。
- 员工到岗和班级负荷。
- 本园区收费、招生、监管缺失。

当前系统已有：

- 单园区维度机构管理、驾驶舱和监管。

当前缺口：

- 集团/多园区数据模型。
- 多园区汇总接口。
- 老板端菜单和驾驶舱。
- 园长与园区的数据范围绑定。

关联任务：T051、T052、T053、T054。

## 11. 建议实施顺序

不要直接继续堆业务页面，建议先把组织和权限主轴立起来：

```text
T048 系统管理后台规划与入口
T049 RBAC 角色权限模型
T051 多园区组织模型设计
T053 园区负责人任命与数据范围控制
T054 园长工作台与园区数据范围
T057 教师班级授权与菜单边界
T070 家长入托关系绑定
T074 入托流程重构
T078 家长手机号注册与短信验证
T079 家长邀请码绑定宝宝和入托档案
```

这样后续每个功能都能明确：

```text
谁创建
谁审核
谁查看
谁修改
谁承担责任
数据属于哪个园区
是否能被老板跨园区查看
是否能被家长查看
是否进入监管材料
```
